# Peer Review Report — Canonical Structure

This is the canonical section structure for the .docx generated in Phase 4. Follow it exactly
so every review has the same shape regardless of manuscript. Bracketed text is a fill-in.
Styling: Arial 12pt body, navy (#1a3a5c) headings, table header rows light-blue (#e8f0f8)
fill, major concerns in bold. Target 3-5 pages excluding the Audit Log.

---

## Header block (not numbered)

- **Manuscript title:** [title]
- **Date of review:** [date]
- *This review was prepared with the assistance of Consensus (consensus.app), which was used
  to search peer-reviewed literature for claim verification, citation completeness, and
  methods assessment. All search results are documented in the Audit Log.*

## 1. Summary

[2-3 sentences: what the paper does, what it claims to show, one sentence of overall
impression. This is what the editor reads first — accurate and specific.]

## 2. Background and Claims

For each key claim checked:
- **Claim (as written):** [author's claim]
- **What Consensus found:** [supporting / contradictory / inconclusive — with result links]
- **Verdict:** Accurate and well-supported / Partially supported — important nuance missing /
  Contradicted by recent literature / Unverifiable with available searches

[If a claim is accurate, say so — reviewers who only flag problems give an incomplete picture.]

## 3. Missing Citations

[Only papers confirmed ABSENT from the Phase 1 reference list (matched on first-author+year or
title) AND genuinely relevant. If no reference list could be extracted, state that citation
completeness could not be assessed rather than listing candidates.]

For each:
- [Full citation] — [Consensus URL]
- *Relevance:* [one sentence on what the authors should engage with it on]

[If nothing important was missed, say so explicitly.]

## 4. Methods Assessment

For each method/approach:
- Current standard, or superseded? [with Search 4 citations]
- Known limitations the authors should acknowledge?
- Do evaluation metrics / endpoints match field standard?

## 5. Major Concerns

Numbered. Each: clear statement of the problem · why it matters for the conclusions · specific
remedy. (1-4 items — more usually signals padding.)

## 6. Minor Concerns

Numbered, same format, for issues that do not threaten the conclusions. (2-6 items.)

## 7. Recommendation

One of: Accept / Minor revision / Major revision / Reject with invitation to resubmit / Reject.
One sentence of justification tied to the Major Concerns above. No long hedging.

## 8. Audit Log

| # | Query | Filters | Results Received | Purpose |
|---|-------|---------|------------------|---------|
| 1 | [query] | [year_min etc.] | [N] | Field consensus |
| 2 | [query] | | [N] | Claim verification |
| ... | | | | |

- Detected Consensus plan tier and per-query cap: [tier / cap]
- Searches that failed and were not recovered: [list, or "none"]
- Self-check (Phase 3.5) removed: [dropped citations / downgraded concerns, or "nothing"]
- *Papers cited in this review were drawn only from Consensus results returned in this session.
  Papers not returned by Consensus are not cited regardless of prior-knowledge relevance.*
